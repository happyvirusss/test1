1.หน้าที่ของ ROS Master
master ทำหน้าที่เป็น server สำหรับให้ node หลายๆตัวเชื่อมต่อกัน และสามารถส่ง message หากันได้ คำสั่งที่ใช้รัน master คือ roscore 
ที่เราใช้ก่อนหน้านี้นี่เอง เมื่อเรารัน master แล้ว เราก็สามารถที่จะสร้าง node ต่างๆขึ้นมาเพื่อดึงข้อมูลได้ การเชื่อมกันระหว่าง node จะส่งข้อความหากันผ่าน topic

2.การทำงานของ ROS Topics
เป็นตัวที่ใช้ทำงาน steam data ถูกใช้งานจากการที่พยายามจะอ่านค่าจากsensorหรืออ่านค่าอะไรต่างๆ
และต้องการนำค่านั้นมาใช้งานต่อ จึงใช้เพื่อเป็นตัวกลาง

3.การท างานของ ROS Service
service เป็น synchronous โดยจะมี service client กับ service server การทำงานก็คือ node ที่เป็น service client 
จะส่งไปหา node service server แล้ว node server ก็จะตอบคำตอบกลับไปที่ node client

4.ความแตกต่างของ Topic กับ message
topicคือ ข้อความจะถูกส่งผ่านระบบการขนส่งที่มีการ  publish / subscribe  โหนดที่ส่งข้อความจะเรียกว่า publishing ชื่อที่ระบุใน Topic 
จะใช้เพื่อระบุเนื้อหาของข้อความ โหนดที่สนใจในข้อมูลบางประเภทจะทำการ subscribe หัวข้อที่ต้องการ
และ message โหนดสื่อสารกันโดยส่งข้อความ โดยอาจจะคุยในรูปแบบของข้อความ ค่ามาตรฐาน ข้อมูลอาร์เรย์ทั่วๆไป รวมไปถึงข้อมูลอาร์เรย์ที่มีความซับซ้อน 

5.อธิบายความแตกต่างในเชิงการใช้งานของ ROS Topic กับ ROS Service
Ros topicจะระบุเนื้อหาของข้อความ โหนดที่สนใจในข้อมูลบางประเภท ส่วนRos service จะสร้างข้อความหนึ่งสำหรับการร้องขอ
และอีกหนึ่งสำหรับการตอบกลับ เพื่อส่งข้อความที่ร้องขอไปยังโหนดที่ร้องขอ
